---
title: When to Conduct an Assessment
description: Timing guidance and institutional triggers for initiating ARAF assessments.
---

The right assessment timing depends on lifecycle stage and institutional pressure. There is no minimum maturity requirement. Organisations can be assessed at any stage, and `ARAF Assessed` requires no minimum GBI score.

## Proactive Triggers

Conduct assessment before pressure arrives where possible:

- Before a funding round
- Before insurance renewal or new coverage placement
- Before enterprise procurement responses
- Before expected regulatory engagement
- At production deployment of a new autonomous system
- When the autonomous system portfolio reaches material scale

Proactive assessments provide remediation time before diligence, underwriting, procurement, or supervision windows open.

## Reactive Triggers

Reactive assessment is still valid where an event has already occurred:

- Adverse outcome where decision reconstruction is not possible
- Insurer request for governance evidence that cannot be produced
- Enterprise customer requiring governance certification
- Regulatory inquiry into AI governance architecture
- Investor diligence flagging governance uncertainty

Reactive assessment is typically costlier because remediation is performed under active scrutiny.

## Reassessment Timing

After initial assessment:

- Maximum interval between formal reassessments is 12 months
- Material changes trigger reassessment regardless of schedule

Material changes include significant model updates, new deployment contexts, material changes to data sources, and material changes to contract infrastructure.

## Timing Principle

If governance architecture affects institutional decisions in your current quarter, assess now. The highest-risk pattern is waiting until a counterpart demands evidence that does not yet exist.
