ARAF is a governance standard for autonomous decision infrastructure. For insurers, its function is practical: convert governance quality from narrative assertion into a comparable underwriting signal.

This page maps current ARAF governance controls to insurer diligence needs, including entity verification, version integrity, route integrity, reassessment timing, and evidence reliability.

Last updated: March 2026.

## Insurer Use Case

Insurers require a governance signal that can be used in:

- pre-bind underwriting diligence
- renewal condition setting
- claims defensibility review
- portfolio-level risk segmentation

ARAF provides this through dimensional scoring, GBI/GCI thresholds, evidence quality tiers, and certification lifecycle controls.

## Underwriting Intake Checklist

Use this checklist for each in-scope autonomous deployment:

- Record the assessed URL and explicit ARAF version in the underwriting file.
- Confirm maintainer entity reference aligns with Institute for Autonomous Governance Pty Ltd (ACN 696 112 277).
- Capture GBI and, where applicable, GCI values with assessment date.
- Capture D1, D3, D4, and D6 profiles for exposure architecture review.
- Record primary evidence-tier profile supporting the assessment.
- Record reassessment obligations, trigger events, and notification window.
- Record whether the certification route cited is canonical or legacy.

## Reliance Boundaries

ARAF certification provides contemporaneous governance evidence. It does not create a statutory safe harbor and does not replace legal, regulatory, or policy obligations in any jurisdiction.

Insurer reliance should therefore treat ARAF as:

- a governance classification signal
- an evidence-quality signal
- a lifecycle-monitoring signal

and not as a substitute for policy-specific underwriting judgment.
