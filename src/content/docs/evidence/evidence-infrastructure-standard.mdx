# Evidence Infrastructure Standard (EIS-01)

**Version 3.0 · March 2026 · Institute for Autonomous Governance Pty Ltd**

## 1. Canonical Properties

The minimum structure of a compliant governance decision record under ARAF must satisfy four properties: authenticity, integrity, traceability, and chain of custody.

## 2. Evidence Tier Requirements

The evidence infrastructure standard defines four evidence tiers:

| Tier | Description | Integrity Anchor |
|------|-------------|-----------------|
| 1    | Infrastructure-generated, tamper-evident records | Required |
| 2    | Contemporaneous documentation | Optional |
| 3    | Reconstructed documentation | Not admissible for certification |
| 4    | Management representation | Excluded |

## 3. D6 Two-Layer Model

The D6 model distinguishes between design evidence and operational evidence. Both must be present for certification.

## 4. Four-Component Canonical Record

A compliant governance decision record must include:
- Execution Event
- Contextual State
- Authority and Attribution
- Integrity Anchor

## 5. Certification Tier Requirements

| Tier | Evidence Requirement |
|------|---------------------|
| Assessed | Tier 1 or Tier 2 |
| Compliant | Tier 1 only |
| Certified | Tier 1 only |

## 6. Governance Agent Output Admissibility

Governance agent outputs are admissible only if:
- Condition 1: Output is produced by a governance agent operating under a defined protocol
- Condition 2: Output is accompanied by a contemporaneous integrity anchor

| Condition | Tier Classification |
|-----------|--------------------|
| Both satisfied | Tier 1 |
| Condition 1 only | Tier 2 |
| Condition 1 not satisfied | Excluded |

## 7. Scope Limitation Requirement

All governance decision records must include a scope limitation statement.
